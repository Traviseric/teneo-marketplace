# Specs

This directory holds task and feature specs for meaningful work in this repo.

Purpose:

- keep implementation intent separate from execution notes
- give future sessions a durable place for scoped build plans
- align the repo with the `BEST_PRACTICES.md` expectation that repos maintain a `specs/` directory

Suggested usage:

- one spec per meaningful feature, audit, migration, or stabilization effort
- keep specs concrete and evidence-based
- link finished work back into `CHANGELOG.md`, `docs/ROADMAP.md`, and `docs/reference/MARKETPLACE_STATUS_AND_TODO.md` when status changes
