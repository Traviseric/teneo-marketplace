## Step 3: Reconcile

For each item in the roadmap:
- Already done (appears in git log or checked off in tasks) -> mark as complete
- In-flight (has an active/*.md task) -> preserve, don't duplicate
- Not started -> add to task list with appropriate priority
- Non-roadmap task from audits/QA/human triage/discovered work -> preserve under `## Preserved Non-Roadmap Work`
- Stale task -> remove ONLY if it is clearly obsolete, duplicated, explicitly superseded, or already completed elsewhere

Do NOT remove a task simply because it does not appear in the roadmap. Roadmaps are not the only valid source of work.

Priority ordering:
- P0: Items the roadmap marks as critical/blocking/phase-1/MVP
- P1: Items marked as important/phase-2/next
- P2: Nice-to-have, future, stretch goals

- Keep descriptions specific and actionable (not vague aspirations)
- Preserve any in-flight active/ tasks by including them in the appropriate section
- Preserve valuable non-roadmap work instead of deleting it
